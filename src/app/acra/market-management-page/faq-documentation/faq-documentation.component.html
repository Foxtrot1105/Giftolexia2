Frequently Asked Questions

A. Getting Started

How do I start using the APIs ?

You may wish to reference to the Getting Started guide on ACRA API Mall portal (https://www.apimall.acra.gov.sg).

Customers who wish to use ACRA API services will first need to complete registration in ACRA API Mall portal using SingPass or CorpPass log in. There is no fee for account registration.

Successfully registered users will be issued with a user Client_ID and Client_Secret key. Customers may subsequently generate authentication token using the assigned Client_ID and Client_Secret key, coupled with customers' Client_name (application name provided by customer). Note that the generated authorisation token expires after 60 minutes. Customers may re-generate the authentication token when needed.

For use of chargeable API services, registered users will need to make the necessary payment before using the APIs. Note that subscriptions are valid for up to 90 days from the date of purchase, or when purchased API calls are completely consumed, whichever happens earlier.

How does system authenticate my request ?

The security method applied is OAUTH2.

Customers may generate authentication token using the assigned Client_ID and Client_Secret key, coupled with customers' Client_name (application name provided by customer). Note that the generated authorisation token expires after 60 minutes. Customers may re-generate the authentication token when needed.

You may reference to information available within Documention page of respective API services for details on generation of authorisation token.

Why is there an expiry date to my purchased API services ?

Each purchase is valid for use within the stipulated period. The system tracks an expiry date for system optimisation purpose. By ensuring minimal inactive user records in the system, system resources may be free up to optimise system performance for active users.

Can I submit for multiple accounts using the same SingPass or CorpPass ?

Each registered user account is tied to a unique SingPass or CorpPass, which acts as the user ID for account log in.

You may wish to be informed that you may provide multiple Client_Name under your registered account in case you have need for the API services in more than one application.

I did not receive the system notification for my registration status.

Approval for account creation is immediate. Once your account has been created, a system notification email will be sent to your provided email address as confirmation of your account creation.

In case the email is not received, you may wish to look into your junk or spam mail folder to check if our system notification email has been directed there.

Customer may log in to ACRA API Mall using SingPass / CorpPass once the account has been created.

How may I register for the service if I do not have a SingPass or CorpPass ?

Note that registration may only be done through SingPass or CorpPass.

All Singapore citizens, Singapore permanent residents, employment pass holders and dependent pass holders who are at least 15 years old can apply for a SingPass. If you do not have a SingPass yet or have forgotten your SingPass password, you can submit an Online Request to apply or reset your SingPass. Your SingPass will be sent to you by post within 4 working days. If you wish to get your SingPass immediately, you can visit the SingPass counters at any of the CPF offices, Community Development Council (CDC) and selected Community Clubs and Community Centres. For more details on SingPass, please refer to https://www.ecitizen.gov.sg/Pages/default.aspx

CorpPass user account can only be created by CorpPass Administrators, who are in turn appointed by an organisation's Registered Officers. Please refer to www.CorpPass.gov.sg for more information. For registered entities in Singapore, please contact CorpPass team to apply for the required CorpPass under your registered entity.

For Non-Singaporeans, you may contact SingPass / CorpPass team to apply for the required CorpPass which ties to your operating entity overseas. Note that CorpPass for foreign entities is available only in Q4'18.

C. API Usage

What is an API call ?

Everytime the API service is invoked to ACRA's servers from customers' application, is considered an API call.

Purchased API call will only be deducted when a successful API call is invoked.

How to identify whether the API call made will be deducted as a successful API call ?

Reference Documentation page of respective API calls to learn about the specific error code scenarios. The error codes which will not contribute to a successful call (i.e. no deduction made) are as stated within the sample responses on Documentation page.

What happens if my purchased API calls have been expended ?

Limit on API calls, as well as expiry date, applies for chargeable API services. For uncharged API services, subscription is for unlimited use.

Use of the API service/s involved will be discontinued when purchased API calls have been expended. You may purchase new API packages to continue using the API services.

Alternatively, you may make use of the system's Auto Top-Up function which is designed to increase convenience for users. Through this function, user permits ACRA API Mall system to automatically top up your balance API calls, with payment through your assigned default payment method.

I still have un-utilised calls on my subscription expiry date and how can I extend the expiry date ?

Expiry date applies for chargeable API services only.

Extension of expiry date is not possible. Please note to utilise all calls within your subscription before the expiry date. Expiry date and outstanding calls availability may be tracked from your Subscription DetailsSubscription Details after logging in to our ACRA API Mall portal through your SingPass / CorpPass.

I am unable to use the API service suddenly. System returns an error code. Why is this so ?

You are advised to check the error code description.

List of error codes may be referenced from the Documentation page of respective API services.

You may also wish to check on the expiry date of your API calls purchase through your Subscription Details page by logging into your account on ACRA API Mall using your SingPass / CorpPass.

How may I test the API services ?

You may utilise the sandbox environment for API testing through the respective API Test Console. The test data which may be used for the sandbox can be found in the API documentation pages.

Customer may select the API category containing the API service of interest, click on the selected API service to view the Documentation page, or click on API Test Console to access the sandbox environment for the API service.

APIs sandbox environment is available for all users. Registration through SingPass or CorpPass, or acocunt log in is not required.

Where can I find the specifications of the APIs ?

Information of the APIs can be found on the Documentation page of the respective API services.

Customer may select the API category containing the API service of interest, click on the selected API service to view the Documentation page.

I have earlier purchased for service of an API category but why am I unable to use the APIs in the category.

You are advised to verify that the purchased API category is displayed through your Subscription Details page by logging into your account on ACRA API Mall using your SingPass / CorpPass. Check on the expiry date of your API calls purchase as well as No. of Remaining API Calls.

Note that of purchased service must not have expired, and that No. of Remaining API Calls must not be zero, in order to be eligible for using the API services.

Otherwise you may also reference the received error code found under Documentation page of respective APIs to learn more on the error code representation.

The data received seems to be incorrect. How may I verify or report the suspected issue ?

For report on API services issue, please contact ACRA Helpdesk at +65 6248 6028, or email to feedback@askacra.sg

I would like to share some feedback on the APIs. How may I submit the feedback to ACRA ?

For comments or feedback on API services, please submit here. Your feedback on our service is much appreciated for our services improvements.

Which API can I use to pull list of all ACRA registered entities ?

This is not possible. Our APIs are designed to retrieve information of unique entity.

How do I differentiate between chargeable or non-chargeable APIs ?

Fees of API categories are displayed under API categories

For API services which are non-chargeable, these may be used without limit once customer subscribes to the category. No payment process is needed. After subscription, customer should be able to view the subscribed category from Subscription Details page after logging into ACRA API Mall portal.

Chargeble API calls may be made after successful payment for the required API categories. After logging into ACRA API Mall, customer may select and add required API categories to cart, and proceed with payment. Once successful payment is completed, the purchased API calls will be reflected on Subscription Details page after logging into ACRA API Mall portal.

Why is the response returning an empty result ( E.g. "entities": [] )?

This response ( "entities": [] ) indicates that the system has performed the search with user's provided inputs, with no results found.

For example, in the API service for Company Charges Details, in case where the provided UEN has no charge information, this empty array response will be returned.

This could also be for instances where, an UEN of a non-company entity type format is provided when the API service triggered is applicable only for company entity type.

How may I identify the UEN format for different business entity types?

UEN format may be referenced from UEN website, About UEN.

D. Account Details and Payment Issues

How can payment be made ?

Payment can be submitted online through the following credit / debit cards:

- MasterCard
- Visa

The above are the only payment modes accepted for API service calls purchase.

Can I make payment using Deposit Service Account (DSA) that I have set up with ACRA ?

The system is unable to accept payment for API subscription using DSA. ACRA API Mall only accepts online payment through valid credit or debit cards (MasterCard / Visa) issued by banks.

Can I use giro for payment?

The system is unable to accept payment for API package purchase using giro. ACRA API Mall only accepts online payment through valid credit or debit cards (MasterCard / Visa) issued by banks.

How may I request for a refund of submitted payment, or a pro-rated refund on un-utilised API calls which I have paid when subscribing to the API package ?

Refund requests for successful payments made cannot be serviced, whether in full or pro-rated. This non-refundable term has been stated as part of the purchase plan, with reminder note before customer proceeds to payment.

Please see below extracted:

Important Note: By choosing to proceed with payment, I agree that the items in the cart have been checked and verified to be correct. These are valid for use within 90 days from this date of purchase, non-refundable and non-transferable.

How may I request for a transfer of balance API calls to another account ?

Requests for transfer of balance API calls cannot be serviced, whether in full or pro-rated. This non-transferable term has been stated as part of the purchase plan, with reminder note before customer proceeds to payment.

Please see below extracted:

Important Note: By choosing to proceed with payment, I agree that the items in the cart have been checked and verified to be correct. These are valid for use within 90 days from this date of purchase, non-refundable and non-transferable.

How do I automate API calls top-up upon calls depletion ?

You may enable Auto Top-Up function available on Subscription DetailsSubscription Details after logging into the ACRA API Mall portal using your SingPass / CorpPass. By confirming the enabling of Auto Top-Up feature, you are granting permission for ACRA API Mall system to automatically top-up your depleted API call balance through your assigned default payment method.

For changes on default payment method, please visit Payment Settings Under My Account menu after logging into.

Auto Top-Up function is disabled by default.

How may I disable the Auto Top-Up function to revent package re-purchase upon calls depletion ?

Auto Top-Up function can be disabled from Subscription DetailsSubscription Details after logging into ACRA API Mall portal using your SingPass / CorpPass.

Auto Top-Up function is disabled by default.

I have opted for Auto Top-Up but why am I having zero API balance calls ?

In case where user opted for Auto Top-Up but there were API calls which were left un-utilised after validity period, hence expired, the Auto Top-Up function will be disabled by system to avoid further payment by user.

If user's balance API calls were only partially expired (i.e. user has API calls with varying expiry dates within the same API category), Auto Top-Up function will not be disabled.

It is posisble for user to re-purchase API package with Auto Top-Up after previous purchase has expired.

E. Technical Setup

Is there any system requirements for using the APIs ?

There is no specific system requirements for using the APIs as long as it is reliable and has active connections to internet.

How may I retrieve or edit my Client_ID and Client_Secret key or Authorisation token ?

Customer needs to be logged in to ACRA API Mall. Click on API Authorisation under My Account menu.

For first time user, click on Add New. The pop up will display the Client_ID and Client_Secret key. Customer may provide the Client_Name and click on Generate token.

Note that Client_ID and Client_Secret key is assigned to unique and is fixed. However, customer may edit Client_Name which may have been created in the past.
